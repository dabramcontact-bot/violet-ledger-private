# Violet Ledger Private

Приватный многопользовательский реестр запросов китайским агентам. Интерфейс публикуется через GitHub Pages, а авторизация, общая база и права пользователей работают через Supabase.

## Безопасность

- Незарегистрированный пользователь не получает данные.
- Доступ появляется только для email, заранее добавленного администратором.
- Роли: `admin`, `editor`, `viewer`.
- Все таблицы защищены Row Level Security.
- В браузере используется только publishable key. Никогда не добавляйте `service_role` key в код или GitHub.

## Подключение Supabase

1. Создайте проект Supabase.
2. Откройте SQL Editor и выполните `supabase/schema.sql`.
3. Первый администратор уже указан в конце SQL-файла.
4. Для локальной работы создайте `.env` по примеру `.env.example`.
5. В настройках репозитория выберите Pages → Source → GitHub Actions.

## Локальный запуск

```bash
npm install
npm run dev
```

## Публикация

Push в ветку `main` запускает `.github/workflows/pages.yml`. Собранный интерфейс не содержит паролей и service-role ключей; доступ к данным проверяется политиками Supabase.
