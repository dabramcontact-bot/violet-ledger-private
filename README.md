# Violet Ledger Private

Приватный многопользовательский реестр запросов китайским агентам. Интерфейс публикуется через GitHub Pages, а авторизация, общая база и права пользователей работают через Supabase.

## Возможности

- сохранение и ведение товарных запросов, предложений и документов PI;
- статистика по китайским агентам, категориям и количеству товаров;
- визуальный маршрут «Китай → агент → логистика → склад»;
- отдельный контроль логистики: статус «ожидает / в пути / на складе», перевозчик и даты;
- независимый раздел платежей, связанный с заказами;
- автоматический расчёт срока оплаты и даты подачи международного платежа;
- предоплата, отсрочка, частичная и полная оплата, остаток и подтверждающие документы;
- напоминания о ближайших и просроченных платежах;
- роли `admin`, `editor` и `viewer`;
- управление приглашениями и ролями прямо в интерфейсе сайта;
- журнал действий по запросам и отдельная история изменений платежей.

## Что защищено

- Незарегистрированный пользователь не получает рабочие данные.
- Регистрация разрешается только для email, заранее добавленных администратором.
- Роли: `admin`, `editor`, `viewer`.
- Таблицы защищены Row Level Security (RLS).
- В браузере используется только безопасный publishable key. `service_role`, secret key и пароль базы нельзя добавлять в GitHub.

## Первый запуск Supabase

1. Откройте `supabase/schema.sql`.
2. Скопируйте весь SQL в Supabase → SQL Editor → New query.
3. В последнем блоке замените `REPLACE_WITH_ADMIN_EMAIL` на email первого администратора в нижнем регистре.
4. Нажмите Run один раз.
5. В Violet Ledger выберите «Получили приглашение? Создать аккаунт» и зарегистрируйтесь с тем же email.

## Обновление существующей базы до режима «только администратор»

Если первоначальная схема уже была установлена, один раз выполните файл `supabase/admin-only-migration.sql` в Supabase SQL Editor. После этого пользователи с режимом просмотра физически не смогут добавлять, изменять или удалять запросы через API.

## Добавление логистики в существующую базу

Выполните `supabase/logistics-upgrade.sql` в Supabase SQL Editor. Файл добавляет логистические поля и одновременно включает admin-only RLS, поэтому отдельный `admin-only-migration.sql` после него выполнять не требуется.

## Добавление платежей в существующую базу

Один раз выполните `supabase/payments-upgrade.sql` в Supabase SQL Editor. Файл создаёт:

- таблицу `payments`;
- связь платежа с заказом через `request_id` и номер запроса;
- плановые и фактические суммы;
- даты документа, оплаты и подачи платежа;
- отсрочку и резерв на международный перевод;
- статусы платежа;
- отдельный журнал `payment_audit_log`;
- RLS-политики и Realtime-обновления.

После выполнения SQL обновите сайт через `Ctrl + Shift + R` и откройте пункт «Платежи» в верхнем меню.

## GitHub Pages

В репозитории откройте Settings → Pages и установите Source: GitHub Actions. После push в `main` workflow `.github/workflows/pages.yml` автоматически собирает и публикует приложение.
