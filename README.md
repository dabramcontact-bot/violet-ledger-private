# Violet Ledger Private

Приватная система закупок с четырьмя самостоятельными рабочими контурами:

`Запрос → PI → Логистика → Платежи`

Интерфейс публикуется через GitHub Pages, а авторизация, общая база и права пользователей работают через Supabase.

## Текущая структура

### 1. Запросы

Самостоятельный реестр запросов поставщикам. Запрос не означает покупку, поэтому логистика и платежи сюда не подтягиваются.

Цикл запроса:

`Запрос отправлен → Предложение получено → Внесено в расчёт`

После этапа «Внесено в расчёт» цикл запроса завершён. В разделе нет PI, результата сделки, операционного маршрута и логистических полей.

### 2. PI

Будет добавлен отдельным этапом позже. Раздел требует самостоятельной схемы и не должен возвращать PI внутрь реестра запросов.

### 3. Логистика

Самостоятельные логистические заявки создаются только для реально купленных позиций. Обязательные поля:

- артикул;
- номер PI;
- стоимость перевозки;
- вид транспорта;
- объём в м³;
- перевозчик;
- поставщик из выпадающего списка.

### 4. Платежи

Независимый раздел платежей по вручную введённому номеру PI. Поддерживает предоплату, баланс, расчёт оплаченной суммы, остатка, срока оплаты и Telegram/email-уведомления.

## Что защищено

- Незарегистрированный пользователь не получает рабочие данные.
- Регистрация разрешается только для email, заранее добавленных администратором.
- Роли: `admin`, `editor`, `viewer`.
- Таблицы защищены Row Level Security (RLS).
- В браузере используется только безопасный publishable key.
- Добавлять `service_role`, secret key и пароль базы в GitHub запрещено.

## Первый запуск Supabase

1. Откройте `supabase/schema.sql`.
2. Скопируйте весь SQL в Supabase → SQL Editor → New query.
3. В последнем блоке замените `REPLACE_WITH_ADMIN_EMAIL` на email первого администратора в нижнем регистре.
4. Нажмите Run один раз.
5. В Violet Ledger зарегистрируйтесь с тем же email.

## Самостоятельная логистика

Один раз выполните файл `supabase/logistics-standalone-upgrade.sql` в Supabase SQL Editor. Скрипт:

- создаёт независимую таблицу `logistics_requests`;
- не связывает логистику с таблицей `requests`;
- включает admin-only редактирование и командный просмотр;
- подключает Realtime;
- безопасен для повторного запуска.

После выполнения обновите сайт через `Ctrl + Shift + R` и откройте пункт «Логистика».

## Платежи по PI

Повторно выполните актуальный файл `supabase/payments-upgrade.sql` в Supabase SQL Editor. Он безопасен для повторного запуска и:

- отвязывает платежи от таблицы запросов;
- добавляет самостоятельный номер PI;
- оставляет только «Предоплата» и «Баланс»;
- рассчитывает оплаченную сумму и остаток;
- рассчитывает дату оплаты как дата документа + отсрочка;
- создаёт настройки уведомлений и журнал отправок;
- сохраняет RLS и Realtime.

## Email и Telegram-уведомления

Исходник функции находится в `supabase/functions/payment-reminders/index.ts`.

Для Telegram нужен секрет `TELEGRAM_BOT_TOKEN`. Для автоматического запуска нужен `PAYMENT_CRON_SECRET`. Ежедневное расписание создаётся файлом `supabase/payment-reminders-schedule.sql`.
